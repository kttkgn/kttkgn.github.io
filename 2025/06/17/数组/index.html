数组

发表于 2025-06-17

1. 核心概念

1.1 数组定义

flowchart TD
  A[数组] --> B[存储方式]
  B --> B1[连续内存空间]
  B --> B2[相同数据类型]
  
  A --> C[访问方式]
  C --> C1["下标索引 O(1)"]
  C --> C2[从0开始]

  A --> D[操作特性]
  D --> D1["插入/删除 O(n)"]
  D --> D2[只能覆盖]

中文解释：

  • 连续内存：数组元素在内存中像排队一样紧密排列，知道第一个元素地址就能快速计算其他元素位置。

  • 下标从0开始：计算机通过首地址 + 下标×元素大小直接定位元素，如arr[2]的地址 = arr + 2*sizeof(int)。

  • 覆盖特性：删除元素本质是后面元素前移覆盖，例如删除arr[3]需要把arr[4]~arr[n]全部左移一位。

2. 内存布局对比

2.1 一维数组

flowchart LR
    A[C++/Go] -->|连续存储| B["0x1000: a[0]\n0x1004: a[1]\n..."]
    C[Python] -->|存储指针| D["0x2000: ptr1→1\n0x2008: ptr2→2"]

中文解释：

  • C++/Go：int arr[3] = {1,2,3}在内存中严格连续，每个int占4字节，地址差为4。

  • Python：list = [1,2,3]实际存储的是指向各个对象的指针（64位系统每个指针占8字节），元素1/2/3可能分散在内存不同位置。

2.2 二维数组

flowchart TB
    subgraph C++/Go
        A["0x1000: a[0][0]] --> B[0x1004: a[0][1]] --> C[...] --> D[0x1010: a[1][0]"]
    end
    subgraph Java/Python
        E[Row0: 0x3000] --> F[Row1: 0x4000]
    end

中文解释：

  • C++：int arr[2][3]在内存中是6个连续的int，按行优先排列。

  • Python：[[1,2],[3,4]]中，第一行[1,2]和第二行[3,4]是两个独立对象，地址无关联。

3. 动态数组扩容

3.1 扩容策略对比

classDiagram
    class 动态数组 {
        <<接口>>
        +append()
        +扩容策略()
    }
    class C_vector {
        +"2倍扩容（可能浪费空间）"
    }
    class Go_slice {
        +"2倍扩容（共享底层数组）"
    }
    class Python_list {
        +"动态扩容（0,4,8,16,25,35...）"
    }
    动态数组 <|-- C_vector
    动态数组 <|-- Go_slice
    动态数组 <|-- Python_list

中文解释：

  • C++：vector容量满时，新分配2倍空间，拷贝旧数据（如原容量4→新容量8）。

  • Go：slice扩容时，若容量不足则新建底层数组（原数据拷贝到新数组）。

  • Python：list采用更复杂的扩容公式，避免频繁扩容（如小列表按4/8/16增长，大列表按1.125倍）。

4. 算法应用示例

4.1 双指针问题

graph TD
    A["双指针"] --> B["快慢指针"]
    A --> C["左右指针"]
    B --> D["删除有序数组重复项"]
    B --> E["环形链表检测"]
    C --> F["两数之和（有序数组）"]
    C --> G["盛水最多容器"]

中文解释：

  • 快慢指针：慢指针指向有效位置，快指针扫描数组。例如删除重复项时，慢指针保留不重复元素。

  • 左右指针：从两端向中间移动。例如计算最大盛水量时，每次移动较矮的指针以寻找更高边界。

4.2 滑动窗口

stateDiagram-v2
    [*] --> 初始化窗口
    初始化窗口 --> 扩大右边界: 扩大窗口（满足条件前）
    扩大右边界 --> 条件满足: 如：子串覆盖目标字符
    条件满足 --> 收缩左边界: 收缩窗口（优化结果）
    收缩左边界 --> 扩大右边界: 继续寻找新解

中文解释：

  • 步骤：先扩展右边界直到满足条件，然后收缩左边界寻找最优解（如最小覆盖子串）。

  • 时间复杂度：O(n)，每个元素最多被左右指针各访问一次。

5. 面试高频问题

5.1 语言特性对比

问题 关键点
“如何实现O(1)时间删除数组元素？” “用末尾元素覆盖目标位置（仅适用于无序数组）”
“Python列表为什么访问快？” “虽然存储指针，但通过索引可直接计算指针位置（O(1)）”
“Go切片扩容后地址是否变化？” “可能变化（新分配底层数组），需注意旧切片的引用问题”

5.2 多维数组应用

journey
    title 二维数组遍历方式
    section 行优先遍历
      用户: 按行访问（适合C++连续存储）, 5
    section 列优先遍历
      用户: 按列访问（可能触发缓存未命中）, 3
    section 螺旋遍历
      用户: LeetCode 54题（控制边界）, 4

中文解释：

  • 行优先：for i in rows: for j in cols，与内存布局一致，缓存命中率高。

  • 列优先：for j in cols: for i in rows，可能频繁跨行访问，性能较差。

  • 螺旋遍历：通过控制上下左右边界实现，注意边界收缩条件。

关键点

  • 二维数组是连续内存：行优先存储。

  • 删除和插入元素效率低：因为需要移动元素。

Java中的数组存储

Java中的数组则有所不同。Java没有暴露指针给程序员，也没有直接控制内存地址的能力。在Java中，二维数组实际上是一个“数组的数组”，因此每一行的数组并不一定存储在连续的内存空间中。每一行实际上是一个指向其他数组的引用。

这些地址不是实际内存地址，而是数组对象的引用，它们指向不同的内存位置，且每行的内存不一定连续。

关键点

  • 二维数组是“数组的数组”，内存上每一行可能不连续。

  • 地址输出是引用的内存地址，而不是元素的实际内存地址。

Go中的数组存储

在Go语言中，数组在内存中是连续存储的，类似于C语言的行为。因此，Go中的数组也是按顺序存储在连续内存空间中的。

然而，Go的数组有一个特殊的地方是它们的大小在定义时是固定的，这意味着一旦创建就不能动态调整大小。如果需要类似动态数组的功能，可以使用切片（slice）。

关键点

  • 数组是连续存储的。

  • 没有内存管理问题，但需要关注切片和数组的区别。

Python中的数组存储

Python的list类型虽然看起来像是数组，但实际上它是动态数组。Python的list是由一组指针构成的，这些指针指向不同的内存区域。因此，Python的list在内存中并不总是连续存储的。

Python中没有内存地址暴露给程序员，因此你不能像C或Go那样直接访问数组元素的内存地址。不过，NumPy中的数组（numpy.array）通常是连续内存的。

关键点

  • list存储不是连续的，而是指向多个不同内存位置的指针。

  • NumPy数组是连续内存，更接近C语言数组。
